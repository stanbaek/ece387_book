Module 0: Setting Up GIT Repositories#

Purpose#

Setup GIT repositories and access on the Master.

Create a repo within the GitHub Classroom:#

  1. Browse to github.com and create a GitHub account if you do not already have one. It is useful if your username is something that identifies you (e.g., bneff1013).

  2. One student per group do the following on your personal computer:

    1. First we are going to setup the repo for the Master. This will allow your instructor to see all of your commits throughout the semester.

    2. Browse to https://classroom.github.com/a/OoH0u_XW

    3. Select ‚ÄúAccept this assignment‚Äù

    4. You may need to hit refresh, but eventually it will provide you a link to the repository.

    5. Browse to your repository.

    6. Note the url for your repository (save this link, it is the best way to check if your repo is updated).

    7. Go to Settings -> Manage access -> and ‚ÄúInvite teams or people‚Äù.

    8. Provide access to your team member using their GitHub user name.

    9. Now we need to do the exact same thing to setup the repo for the robot.

    10. Browse to https://classroom.github.com/a/0MPq4TNE and repeat steps c-h.

Enable SSH connection to your GitHub account#

  1. Open a terminal on your Master (ctrl+alt+t).

  2. The same student as step 1.1.2 do the following:

    1. Generate a new SSH key, substituting your GitHub email address:

      ssh-keygen -t ed25519 -C "your_email@example.com"

    2. When you‚Äôre prompted to ‚ÄúEnter a file in which to save the key,‚Äù click enter.

    3. At the prompt, type a secure passphrase.

    4. Start the ssh-agent in the background and add your SSH private key to the ssh-agent:

      eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

    5. Open the public key:

      nano ~/.ssh/id_ed25519.pub

    6. Select the contents of the file (maximize the window and ensure it has your GIT email at the end), right click, and select copy.

    7. Open a web browser and sign in to your GitHub account.

    8. In the upper-right corner of any page, click your profile photo, then click Settings.

    9. In the user settings sidebar, click SSH and GPG keys.

    10. Click New SSH key

    11. In the ‚ÄúTitle‚Äù field, add a descriptive label for the new key, such as ‚ÄúMasterX‚Äù.

    12. Paste your key into the ‚ÄúKey‚Äù field (contents of the .pub file).

    13. Click Add SSH key.

    14. If prompted, confirm your GitHub password.

    15. Create a secure shell connection to your Robot (password is dfec3141)

      ssh pi@robotX

    16. Repeat steps a-f on your Robot and j-n on your Master.

Clone repository to your master.#

  1. On the Master, open the GitHub repository and copy your repo address using the SSH mode:

  2. Open a terminal and browse to your workspace source folder:

    cd ~/master_ws/src/

  3. Clone your repo using the username and password used when you generated the SSH key, replacing USERNAME with your GitHub username:

    git clone git@github.com:ECE387/ece387_master_spring202X-USERNAME.git

  4. Update your git email address and the last name for you and your team mate.

    git config --global user.email "you@example.com"
git config --global user.name "Lastname1 Lastname2"

Clone repository to your robot.#

  1. Create a secure shell connection to your robot:

    ssh pi@robotX

  2. Ensure you are in the ROS robot workspace src directory.

    cd robot_ws/src

  3. Clone the robot repository:

    git clone git@github.com:ECE387/ece387_robot_spring202X-USERNAME.git

  4. Update your git email address and the last name for you and your team mate.

    git config --global user.email "you@example.com"
git config --global user.name "Lastname1 Lastname2"

Lab 2: Inertial Measurement Unit#

Purpose#

This lab will integrate the on-board IMU with the Turtlebot3 controller to turn the robot 90 degrees left or right. Do not disable any of the mouse controller functionality in the turtlebot_controller.py code. The IMU and mouse control functionality should be able to coexist seemlessly.

Master#

Setup:#

In the /master_ws/src/ece387_master_sp2X-USERNAME/master folder, create a lab2 package which depends on std_msgs, rospy, geometry_msgs, and turtlebot3_bringup.

controller.py#

  1. Copy the turtlebot_controller.py file from lab1 into the lab2 package.

  2. Open the turtlebot_controller.py file from lab2 using the Sublime editor.

  3. Import the squaternion library and Imu message used in ICE6.

  4. Add the following Class variables within the class above the __init__() function:

    1. K_HDG = 0.1 # rotation controller constant

    2. HDG_TOL = 10 # heading tolerance +/- degrees

    3. MIN_ANG_Z = 0.5 # limit rad/s values sent to Turtlebot3

    4. MAX_ANG_Z = 1.5 # limit rad/s values sent to Turtlebot3

  5. Add the following to the __init__() function:

    1. Instance variable, self.curr_yaw, initialized to 0 to store the current orientation of the robot

    2. Instance variable, self.goal_yaw, initialized to 0 to store the goal orientation of the robot

    3. Instance variable, self.turning, initialized to False to store if the robot is currently turning

    4. A subscriber to the IMU topic of interest with a callback to the callback_imu() function

  6. Add the convert_yaw() function from ICE6.

  7. Add the callback_imu() function from ICE6, removing print statements and setting the instance variable, self.curr_yaw.

  8. Edit the callback_controller() function so it turns the robot 90 degrees in the direction inputed by the user (left or right). Below is some pseudo-code to help you code the controller function

        # check input and determine goal yaw
        if input is equal to "l"

            turning equals True
        else if input is equal to "r"
           	set goal yaw to curr yaw plus/minus 90
            turning equals True
        else 
        	print error and tell user valid inputs
            
        # check bounds
        if goal_yaw is less than 0 then add 360
        else if goal_yaw is greater than 360 then subtract 360
    
    # turn until goal is reached
    elif turning:
        yaw_err = self.goal_yaw - self.curr_yaw
        
        # determine if robot should turn clockwise or counterclockwise
        if yaw_err > 180:
            yaw_err = yaw_err - 360
        elif yaw_err < -180:
            yaw_err = yaw_err + 360
            
        # proportional controller that turns the robot until goal

        ang_z = self.K_HDG * yaw_err
